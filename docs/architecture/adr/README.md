---

# 1. 목적

이 폴더는 AI Love Workout 프로젝트의 중요한 설계 결정을 기록한다.

ADR은 단순히 결과를 기록하는 문서가 아니라,
왜 그러한 설계를 선택했는지를 남기는 문서이다.

시간이 지나도 설계 의도를 이해할 수 있도록 작성한다.

---

# 2. 언제 ADR을 작성하는가

다음과 같은 경우 ADR을 작성한다.

* 시스템 구조에 영향을 주는 결정
* 장기적으로 유지될 설계 결정
* 여러 선택지를 비교한 결정
* 추후 변경 비용이 큰 결정
* 면접이나 프로젝트 설명에서 중요한 설계 결정

---

# 3. 언제 작성하지 않는가

다음과 같은 내용은 ADR로 작성하지 않는다.

* 단순 버그 수정
* 코드 리팩토링
* 변수명 변경
* UI 수정
* 일시적인 구현 방법

이러한 내용은 Journal 또는 Git Commit으로 관리한다.

---

# 4. ADR 작성 원칙

하나의 ADR은 하나의 결정만 다룬다.

ADR은 구현 방법보다
설계 이유를 설명한다.

ADR은 시간이 지나도 이해할 수 있도록 작성한다.

---

# 5. ADR 번호 규칙

ADR은 다음 형식을 따른다.

ADR-001

ADR-002

ADR-003

...

번호는 생성 순서를 의미한다.

삭제하지 않고 Deprecated 상태로 관리한다.

---

# 6. ADR 템플릿

모든 ADR은 다음 형식을 따른다.

* 제목
* 상태
* 작성일
* 배경
* 문제
* 고려한 선택지
* 최종 결정
* 결정 이유
* 영향
* 관련 문서
