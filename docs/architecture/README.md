---

# 1. 목적

이 폴더는 AI Love Workout의 시스템 설계를 관리한다.

요구사항을 어떻게 구현하기로 설계했는지를 기록하며,
장기적으로 유지되는 설계 문서를 관리한다.

---

# 2. 포함하는 문서

이 폴더에는 다음과 같은 문서를 포함한다.

* ADR (Architecture Decision Record)
* 시스템 아키텍처
* 도메인 모델
* ERD
* 시퀀스 다이어그램
* 컴포넌트 다이어그램
* 배포 구조
* API 설계 문서

---

# 3. 포함하지 않는 문서

다음 문서는 architecture에서 관리하지 않는다.

* 요구사항
* 회의 기록
* 학습 기록
* 개발 일지

각 문서는 자신의 폴더에서 관리한다.

---

# 4. ADR

중요한 설계 결정은 ADR로 관리한다.

ADR에는 다음 내용을 포함한다.

* 배경
* 문제
* 고려한 선택지
* 최종 결정
* 결정 이유
* 영향

---

# 5. 문서 작성 원칙

* 요구사항보다 하위 계층이다.
* 구현 코드보다 상위 계층이다.
* 구현이 변경되어도 설계 의도가 유지되도록 작성한다.
* 특정 프레임워크보다 설계 원칙을 우선한다.
