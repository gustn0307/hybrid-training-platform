# AI Love Workout 사용자 스토리

## 1. 문서 정보

* 프로젝트: AI Love Workout
* Sprint: Sprint 0
* 상태: APPROVED
* 작성일: 2026-07-15
* 작성자: 강현수

## 2. 문서 목적

이 문서는 AI Love Workout MVP의 사용자 가치를 Epic과 사용자 스토리 단위로 정의한다.

각 사용자 스토리는 이후 기능 요구사항, API, 데이터 모델, 화면과 테스트 케이스로 추적할 수 있도록 고유 ID를 사용한다.

## 3. 사용자 스토리 작성 원칙

* 사용자 관점에서 작성한다.
* 구현 기술을 포함하지 않는다.
* 하나의 명확한 사용자 가치를 표현한다.
* 완료 여부를 검증할 수 있는 인수 조건을 포함한다.
* MVP 범위와 MoSCoW 우선순위를 명시한다.
* 주 페르소나의 핵심 문제 해결을 우선한다.

# EPIC-01 사용자 인증

## US-01-01 이메일 회원가입

### 사용자 스토리

나는 처음 서비스를 사용하는 사용자로서 이메일, 비밀번호와 닉네임으로 가입하고 싶다. 왜냐하면 내 운동 정보와 기록을 개인 계정에 안전하게 저장하고 싶기 때문이다.

### 우선순위

Must

### 인수 조건

```text
Given 가입하지 않은 이메일을 사용하는 사용자가
When 유효한 이메일, 비밀번호와 닉네임을 입력해 회원가입하면
Then 사용자 계정이 생성되어야 한다.
And 사용자는 로그인하거나 로그인 상태로 전환될 수 있어야 한다.
```

```text
Given 이미 가입된 이메일이 존재할 때
When 같은 이메일로 회원가입을 시도하면
Then 계정을 중복 생성하지 않아야 한다.
And 이메일이 이미 사용 중임을 안내해야 한다.
```

# EPIC-03 운동 계획 후보

### 우선순위

Must

### 인수 조건

```text

Then 입력한 시간이 운동 계획 후보 생성 조건에 포함되어야 한다.
```

```text
Given 사용자가 허용되지 않는 시간 값을 입력했을 때
```

Must

```text
And 후보의 예상 총 소요 시간은 사용자가 입력한 가용 시간을 초과하지 않아야 한다.

```

```text
Given 저장되지 않은 입력 안내가 표시된 상태에서
When 사용자가 세션 종료를 취소하면
Then 운동 세션은 진행 중 상태를 유지해야 한다.
```

### 중요한 용어 구분

여기서 “세션 종료를 취소한다”는 의미는 세션을 `CANCELLED` 상태로 만든다는 뜻이 아니다.
세션 종료 요청을 철회하고 현재 운동 세션과 작성 중인 입력 흐름을 계속 진행한다는 의미이다.

# EPIC-07 운동 기록 조회 및 관리

## US-07-01 날짜별 운동 세션 조회

### 사용자 스토리

### 우선순위

Must

### 인수 조건

```text

When 날짜별 운동 내역을 조회하면
Then 자신의 운동 세션만 표시되어야 한다.
And 각 세션의 시작 시각, 종료 시각과 총 세션 시간이 표시되어야 한다.
```

## US-07-02 운동 세션 상세 조회

### 사용자 스토리

나는 사용자로서 하나의 운동 세션에 포함된 개별 운동 기록을 확인하고 싶다. 왜냐하면 한 번의 운동에서 수행한 러닝과 근력운동 내용을 함께 보고 싶기 때문이다.

### 우선순위

Must

### 인수 조건

```text
Given 자신의 운동 세션이 존재할 때
When 세션 상세를 조회하면
Then 세션의 시작 시각, 종료 시각과 총 세션 시간을 확인할 수 있어야 한다.
And 저장된 운동 기록이 있다면 해당 기록이 표시되어야 한다.
And 다른 사용자의 세션은 조회할 수 없어야 한다.
```

Must

### 인수 조건

```text
Then 운동 유형에 맞는 상세 필드가 표시되어야 한다.
And 다른 사용자의 기록은 조회할 수 없어야 한다.
```

## 6. 검증이 필요한 사항

* 운동 계획 후보에서 사용자가 수정할 수 있는 필드 범위
* 운동 가능한 시간의 최소값과 최대값
* 최근 운동 기록으로 사용할 기간
* 운동 기록 삭제 방식
* 인증 만료 시 입력 내용 보존 방식
* 자연어 원문 저장 여부
* AI 구조화 결과의 수정 이력 저장 여부
* 종료되지 않은 운동 세션의 자동 처리 방식
* 운동 중 입력한 자연어 또는 구조화 결과의 임시 보존 방식
* 세트 사이 여러 번 입력한 결과를 하나의 운동 기록으로 합치는 방식
* 기록 없이 종료된 세션의 운동 내역 화면 노출 방식
* 미완료 입력 폐기 시 추가 확인 필요 여부
