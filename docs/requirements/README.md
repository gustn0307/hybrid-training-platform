# AI Love Workout 요구사항

## 1. 문서 목적

이 디렉터리는 AI Love Workout 프로젝트의 제품 및 시스템 요구사항을 관리한다.

구현할 기능을 단순히 나열하는 것이 아니라 다음 내용을 명확히 정의하는 것을 목적으로 한다.

* 해결하려는 사용자 문제
* 주요 사용자
* 서비스가 제공할 핵심 가치
* 기능 요구사항
* 비기능 요구사항
* MVP 범위
* 제외 범위
* 성공 지표
* 미확정 사항

## 2. 요구사항 원칙

1. 모든 주요 기능은 사용자 문제와 연결되어야 한다.
2. 기술 선택보다 사용자 요구사항을 먼저 정의한다.
3. 요구사항은 테스트하거나 확인할 수 있어야 한다.
4. MVP에 포함되는 기능과 제외되는 기능을 명확히 구분한다.
5. AI 결과는 오류 가능성을 고려해 사용자가 확인하거나 수정할 수 있어야 한다.
6. 운동 및 건강 관련 데이터는 보안과 개인정보 보호를 우선한다.
7. 웨어러블 연동은 특정 업체에 강하게 종속되지 않는 구조로 설계한다.
8. 요구사항이 변경되면 변경 이유와 영향을 문서에 남긴다.

## 4. 문서별 책임

- 핵심 사용자 문제와 제품 가설의 기준 문서는 `problem-definition.md`이다.
- MVP 포함 범위와 제외 범위의 기준 문서는 `mvp-scope.md`이다.
- 기능별 상세 동작과 인수 조건의 기준 문서는 `functional-requirements.md`이다.
- 성능, 보안, 가용성 등의 품질 기준은 `non-functional-requirements.md`에서 관리한다.
- 성공 지표의 계산 방식과 목표값은 `success-metrics.md`에서 관리한다.

## 5. 요구사항 상태

요구사항은 다음 상태로 관리한다.

* `DRAFT`: 논의 중인 초안
* `APPROVED`: 구현 대상으로 승인
* `IN_PROGRESS`: 구현 진행 중
* `DONE`: 구현 및 검증 완료
* `DEFERRED`: 이후 버전으로 연기
* `REJECTED`: 구현하지 않기로 결정

## 6. 요구사항 우선순위

우선순위는 MoSCoW 방식을 사용한다.

* `Must`: MVP 출시를 위해 반드시 필요
* `Should`: 중요하지만 임시 대안이 존재
* `Could`: 있으면 좋지만 없어도 핵심 가치를 제공 가능
* `Won't`: 현재 범위에서는 구현하지 않음
