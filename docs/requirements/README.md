# AI Love Workout 요구사항

## 1. 문서 목적

이 디렉터리는 AI Love Workout 프로젝트의 제품 및 시스템 요구사항을 관리한다.

구현할 기능을 단순히 나열하는 것이 아니라 다음 내용을 명확히 정의하는 것을 목적으로 한다.

* 해결하려는 사용자 문제
* 주요 사용자
* 서비스가 제공할 핵심 가치
* 기능 요구사항
* 비기능 요구사항
* MVP 범위
* 제외 범위
* 성공 지표
* 미확정 사항

## 2. 요구사항 원칙

1. 모든 주요 기능은 사용자 문제와 연결되어야 한다.
2. 기술 선택보다 사용자 요구사항을 먼저 정의한다.
3. 요구사항은 테스트하거나 확인할 수 있어야 한다.
4. MVP에 포함되는 기능과 제외되는 기능을 명확히 구분한다.
5. AI 결과는 오류 가능성을 고려해 사용자가 확인하거나 수정할 수 있어야 한다.
6. 운동 및 건강 관련 데이터는 보안과 개인정보 보호를 우선한다.
7. 웨어러블 연동은 특정 업체에 강하게 종속되지 않는 구조로 설계한다.
8. 요구사항이 변경되면 변경 이유와 영향을 문서에 남긴다.

## 4. 문서별 책임

- 핵심 사용자 문제와 제품 가설의 기준 문서는 `problem-definition.md`이다.
- MVP 포함 범위와 제외 범위의 기준 문서는 `mvp-scope.md`이다.
- 기능별 상세 동작과 인수 조건의 기준 문서는 `functional-requirements.md`이다.
- 성능, 보안, 가용성 등의 품질 기준은 `non-functional-requirements.md`에서 관리한다.
- 성공 지표의 계산 방식과 목표값은 `success-metrics.md`에서 관리한다.

## 5. 요구사항 상태

요구사항은 다음 상태로 관리한다.

* `DRAFT`: 논의 중인 초안
* `APPROVED`: 구현 대상으로 승인
* `IN_PROGRESS`: 구현 진행 중
* `DONE`: 구현 및 검증 완료
* `DEFERRED`: 이후 버전으로 연기
* `REJECTED`: 구현하지 않기로 결정

## 6. 요구사항 우선순위

우선순위는 MoSCoW 방식을 사용한다.

* `Must`: MVP 출시를 위해 반드시 필요
* `Should`: 중요하지만 임시 대안이 존재
* `Could`: 있으면 좋지만 없어도 핵심 가치를 제공 가능
* `Won't`: 현재 범위에서는 구현하지 않음

## 7. 용어 기준

| 용어 | MVP에서의 의미 |
|---|---|
| 운동 유형 | 러닝, 웨이트, 맨몸운동의 최상위 분류 |
| 운동 종목 | 푸시업, 스쿼트, 벤치프레스와 같은 개별 운동 |
| 근력운동 | 웨이트와 맨몸운동을 포괄하는 상위 개념 |
| 운동 계획 후보 | 사용자가 선택·수정·거절할 수 있도록 시스템이 제시한 확정 전 계획 한 개 |
| 확정 운동 계획 | 사용자가 운동 계획 후보를 그대로 선택하거나 수정한 뒤 확정한 계획 |
| 운동 세션 | 한 번의 운동 시작부터 종료까지의 단위이며, 확정 운동 계획이 있거나 없는 상태로 시작할 수 있고 운동 기록을 포함하지 않거나 하나 이상 포함할 수 있음 |
| 운동 기록 | 진행 중인 운동 세션에 저장되는 실제 수행 개별 운동 데이터 |
| 최근 운동 기록 | 운동 계획 후보 생성 시 조회하는 이전 운동 기록 |
| 운동 강도 | 러닝은 목표 페이스 또는 자유 러닝 여부, 웨이트는 중량·반복 횟수·세트 수, 맨몸운동은 반복 횟수 또는 수행 시간·세트 수로 표현 |
| 거절 | 운동 계획 후보를 사용하지 않고 확정 운동 계획 없이 운동 세션을 시작하거나 현재 흐름을 종료하는 행동 |
| 재생성 | 새로운 운동 계획 후보를 다시 요청하는 행동 |
| 운동 유형별 배정 시간 | 복합 운동 계획에서 각 운동 유형에 배정된 예상 소요 시간 |
| 기록 없는 종료 세션 | 저장된 운동 기록 없이 사용자의 확인을 거쳐 종료된 운동 세션이며 운동 통계와 운동 계획 후보 생성용 최근 기록에서 제외됨 |
