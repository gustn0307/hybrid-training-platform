## 2. 주요 사용자

AI Love Workout의 초기 핵심 사용자는 러닝과 웨이트 또는 맨몸운동을 함께 수행하며, 자신의 운동 기록과 운동 가능한 시간을 바탕으로 다음 운동 계획을 안내받고 싶은 사용자이다.

이 사용자는 전문 트레이너에게 지속적으로 관리받기는 어렵지만, 매번 운동 종류와 강도를 혼자 결정해야 하는 부담을 줄이고 싶어 한다.

## 3. 사용자가 겪는 핵심 문제

러닝과 근력운동을 함께 수행하는 사용자는 자신의 운동 목표, 최근 운동 기록, 현재 수행 능력과 운동 가능 시간을 함께 고려하여 오늘 수행할 운동의 종류와 강도를 결정하기 어렵다.

운동별 애플리케이션은 러닝이나 웨이트 기록에는 특화되어 있지만, 서로 다른 운동 기록을 통합하여 다음 운동 계획을 제안하는 기능은 제한적이다.

## 4. 현재 사용 중인 대안

사용자는 현재 다음 방법으로 문제를 해결한다.

* 운동 일지를 직접 작성한다.
* 러닝 앱과 웨이트 기록 앱을 각각 사용한다.
* 유튜브, 블로그 또는 공개 운동 루틴을 따라 한다.
* 범용 AI 서비스에 자신의 상황을 직접 설명하고 운동을 추천받는다.
* 친구나 운동 경험자에게 조언을 구한다.
* 비용을 지불하고 퍼스널 트레이너에게 지도를 받는다.
* 별도의 계획 없이 기억과 감각에 의존하여 운동한다.

## 5. 기존 대안의 한계

운동 일지를 직접 작성하는 방식은 운동명, 중량, 반복 횟수, 세트 수와 같은 정보를 매번 입력해야 하므로 번거롭다.

운동별 애플리케이션을 여러 개 사용하는 경우 러닝과 근력운동 데이터가 서로 분리되어 전체 운동량과 운동 간 균형을 파악하기 어렵다.

범용 AI 서비스는 사용자의 운동 기록을 운동 데이터 형태로 지속해서 관리하지 않기 때문에 사용자가 질문할 때마다 자신의 상태와 이전 운동 내용을 다시 설명해야 한다.

퍼스널 트레이닝은 전문적인 도움을 받을 수 있지만 지속적인 비용이 발생하고, 사용자가 원하는 시간에 항상 이용하기 어렵다.

## 6. 제품 가설

> 러닝과 근력운동을 함께 수행하는 사용자가 운동 결과를 자연어로 편리하게 기록하고, 서비스가 누적된 운동 기록과 오늘 운동 가능한 시간을 바탕으로 다음 운동 계획 후보를 제안한다면 사용자는 운동 계획을 혼자 결정해야 하는 부담을 줄일 수 있을 것이다.

## 7. 핵심 가치 제안

> 러닝과 근력운동을 함께 수행하는 사용자가 자연어로 운동을 기록하고, 자신의 기록과 일정에 맞는 다음 운동 계획을 안내받을 수 있도록 한다.

서비스를 짧게 표현하는 문구:

> 기록하면, 다음 운동이 보인다.

## 8. AI의 역할

### 8.1 자연어 운동 기록 구조화

AI는 사용자가 일상적인 문장으로 입력한 운동 결과를 다음과 같은 구조화된 운동 기록으로 변환한다.

* 운동 유형
* 운동 종목
* 중량
* 반복 횟수
* 세트 수
* 거리
* 운동 시간
* 사용자 메모

AI가 생성한 결과는 사용자가 확인하고 수정한 뒤 저장한다.

### 8.2 운동 계획 설명

MVP에서는 운동 계획의 핵심 조건과 제한을 운동 규칙이 결정한다.

AI는 다음 역할을 담당한다.

* 사용자의 자연어 요청 해석
* 생성된 운동 계획 후보 설명
* 추천 이유를 사용자가 이해하기 쉬운 문장으로 표현

AI가 사용자 건강 상태를 의학적으로 판단하거나 운동을 강제적으로 처방하지 않는다.

### 8.3 AI 실패 시 대체 방식

AI 분석이 실패하거나 결과가 정확하지 않은 경우 사용자는 일반 입력 화면에서 운동 기록을 직접 입력하거나 수정할 수 있어야 한다.

## 9. 첫날 사용자 경험

1. 사용자가 회원가입한다.
2. 사용자가 주된 운동 목표를 선택한다.
3. 사용자가 오늘 운동 가능한 시간을 입력한다.
4. 사용자가 러닝, 웨이트, 맨몸운동 중 오늘 하고 싶은 운동 유형을 선택한다.
5. 시스템이 입력 정보와 사용 가능한 운동 기록을 바탕으로 운동 계획 후보를 제시한다.
6. 사용자가 운동 계획 후보와 추천 이유를 확인한다.
7. 사용자가 계획을 선택하거나 수정한다.
8. 사용자가 운동 시작 버튼을 누른다.
9. 사용자가 운동을 수행한다.
10. 사용자가 운동 결과를 자연어 또는 일반 입력 화면으로 입력한다.
11. 시스템이 자연어 입력을 구조화된 운동 기록으로 변환한다.
12. 사용자가 운동 기록을 확인하고 수정한 뒤 저장한다.
13. 사용자가 운동 종료 버튼을 누른다.
14. 사용자가 오늘의 운동 시간과 운동 기록을 확인한다.

## 11. 주요 가정

* 사용자는 자연어 입력을 일반 입력 화면보다 편리하게 느낄 것이다.
* 사용자는 러닝과 근력운동 기록을 하나의 서비스에서 관리하는 것을 유용하게 느낄 것이다.
* 사용자는 자신의 최근 기록과 가용 시간을 반영한 운동 계획 후보를 유용하게 느낄 것이다.
* 사용자는 AI가 구조화한 운동 기록을 확인하고 수정하는 과정을 감수할 것이다.
* 사용자는 AI의 운동 계획을 확정적인 처방보다 선택 가능한 제안으로 받아들일 것이다.
