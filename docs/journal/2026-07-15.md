# 2026-07-15 개발 일기

오늘은 AI Love Workout의 사용자 여정과 사용자 스토리를 정의했다.

처음에는 사용자 여정 문서와 사용자 스토리 문서를 제공받은 초안 그대로 생성했지만, 실제 운동 상황을 생각하면서 운동 중 기록 시점과 예외 흐름을 수정했다.

특히 웨이트와 맨몸운동은 세트 사이에 기록할 수 있어야 하고, 러닝은 종료 후 기록하는 흐름이 더 자연스럽다고 판단했다. 또한 운동 기록이 없는 세션도 강제로 막지 않고 사용자 확인 후 종료하되, 통계와 다음 운동 추천에서는 제외하기로 결정했다.

하나의 결정이 사용자 여정뿐 아니라 사용자 스토리, MVP 범위와 용어 정의까지 영향을 준다는 것을 경험했다.

요구사항 작성은 코딩보다 느리고 답답하게 느껴졌지만, 구현 전에 예외와 완료 조건을 정리하면 개발 중 발생할 혼란과 재작업을 줄일 수 있다는 점을 이해했다.

다음 단계에서는 사용자 스토리를 시스템의 기능 요구사항으로 변환할 예정이다.
